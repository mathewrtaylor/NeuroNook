# NeuroNook (NN) GitHub Guides

# Full Novice Guide (Markdown)

## Introduction

Welcome to the NeuroNook (NN) GitHub Guide.

This guide is designed for people with little or no GitHub experience. The goal is to help moderators and volunteers confidently access shared resources, communicate with the team, and navigate repositories without needing technical knowledge.

GitHub can be thought of as a structured online filing cabinet where teams organize documents, resources, templates, and collaborative materials.

This guide focuses only on the web version of GitHub.

---

## What GitHub Is (Without Technical Jargon)

GitHub is an online workspace used to organize files and resources.

Think of it like this:

- The **Organization** is the entire NN workspace
- A **Repository** (“repo”) is a folder for a specific subject area
- A **File** is an individual document
- An **Issue** is a request, question, or problem report

GitHub helps ensure everyone is viewing the most current version of a file.

   - Email address
   - Password
   - Username
5. Complete the verification steps
6. Verify your email address

### Step 2 — Enable Two-Factor Authentication (2FA)

NN requires two-factor authentication for security.

2FA adds an additional security check when logging in. Even if someone knows your password, they still cannot access your account without your second authentication method.

### Recommended Method

Use an authentication app such as:

- Google Authenticator
- Microsoft Authenticator
- Authy

### Setup Steps

1. Click your profile picture
2. Select **Settings**
3. Select **Password and Authentication**
4. Enable **Two-Factor Authentication**
5. Follow the prompts
6. Save your recovery codes somewhere safe

## Using Issues

## Creating an Issue

1. Open the repository
2. Select **Issues**
3. Select **New Issue**
4. Fill out the form
5. Select **Submit**

### Helpful Issue Writing Tips

Good issues explain:

- What the problem is
- Why it matters
- Where it was found
- What change is requested

## Requesting Changes

If you notice something that should be updated:

1. Create an Issue
2. Clearly explain the requested change
3. Include examples if possible
4. Explain why the change would help

The “why behind the what” helps Leads and Admins make informed decisions.

## Accessibility Suggestions

Everyone works differently. GitHub has several helpful accessibility options.

### Helpful Suggestions

- Use browser zoom if text is too small
- Use dark mode if bright screens are uncomfortable
- Use full-screen mode to reduce distractions
- Break large reading sessions into smaller chunks

---

## Security & Privacy Guidelines

### Never Upload

- Personal member information
- Private conversations
- Sensitive moderation discussions
- Passwords or recovery codes

### Why This Matters

GitHub is collaborative by design. Accidentally sharing sensitive information can expose private data to other team members.

---

## Account Recovery

If you lose access to your authentication device:

1. Use your recovery codes
2. Use GitHub password recovery
3. Contact an NN Admin if necessary

## Troubleshooting FAQ

### I Cannot See the NN Organization

- Check your invitation email

### My 2FA Is Not Working

- Verify the time on your phone is correct

- Use repository search
- Ask in Discussions
